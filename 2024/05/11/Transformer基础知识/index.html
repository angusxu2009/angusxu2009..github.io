摘要

主流的序列到序列的模型（基于循环神经网络、卷积神经网络…）通常包括Encoder和Decoder。

性能最好的那些模型，在Encoder和Decoder之间会用注意力机制。

这篇文章提出了一个简单的架构：Transformer。

结论

1、Transformer：第一个做序列转录的模型、仅仅使用注意力，把以往的循环神经网络全都换成了多头自注意力。

2、在机器翻译任务上比其他模型训练得要更快、实际效果也很好。

3、纯注意力机制模型可以用在其他任务上面。

4、code：https://github.com/tensorflow/tensor2tensor

导言

在时序模型里，当前（2017年）最常用的是RNN、LSTM。

有两个比较主流的模型：语言模型、编码器-解码器模型（输入输出结构化信息比较多）

RNN的特点：假设序列是一个句子，RNN是一个一个词地处理。
把之前的信息放在一个隐状态ht−1h_{t-1}ht−1​，和当前词做计算得到hth_tht​。

缺点：

1、时序模型，串行。

2、远端信息遗忘，为了解决这个问题需要把hth_tht​做的很大，存储历史信息。这样内存开销很大。

Attention在RNN中的应用：如何让编码器到解码器更有效得传输。

背景

如何用卷积神经网络来替换循环神经网络，卷积每次只能看一个小的窗口的数据，而Transformer可以一次看到所有的序列。

卷积的一个好处是可以做多个输出通道，一个输出通道可以去识别不一样的模式，由此启发提出了多头注意力(Multi-Headed Attention)

Self-Attention其实在之前的一些工作有提出，本工作不是独创。

Transformer是第一个只用Attention的Encoder-Decoder模型。

模型架构

序列模型里，比较好的架构是编码器-解码器模型架构。

编码器：假设(x1,x2,...,xt,...,xn)(x_1, x_2, ..., x_t, ..., x_n)(x1​,x2​,...,xt​,...,xn​)是一个序列（句子），xtx_txt​就是第ttt个词，编码器会将这个序列转化为长度同样为nnn的(z1,z2,...,zn)(z_1, z_2, ..., z_n)(z1​,z2​,...,zn​)，其中ztz_tzt​为xtx_txt​对应的向量表示。

解码器：输入为编码器的输出:ZZZ，输出为长度为mmm的序列(y1,y2,...,ym)(y_1, y_2, ..., y_m)(y1​,y2​,...,ym​)。nnn和mmm可以不一样，比如在机器翻译里，输入和输出的句子长度困可能是不一样的。

注意，解码器和编码器有一个不同：解码器是一个一个词的输出（自回归，Auto-Regressive），首先用ZZZ生成y1y_1y1​，拿到y1y_1y1​之后用(Z,y1)(Z,y_1)(Z,y1​)来生成y2y_2y2​。也就是生成yty_tyt​的时候要用到y1,...,yt−1y_1,...,y_{t-1}y1​,...,yt−1​。

Transformer是一个编码器和解码器都堆叠了「自注意力」和「Point Wise」、「全连接层」的结构

[这里放一张Transformer的结构图，包含Encoder和Decoder]

编码器和解码器在图上都有输入和输出，但是实际上解码器在预测的时候是没有输入的，而是解码器在之前时刻的输出作为输入。Shifted right就是一个一个往右边移动。

Nx表示有N个块(Transformer Block)堆叠在一起。

Encoder：X+位置编码 -> 多头自注意力层 -> Add & Norm -> MLP -> Output

Encoder的输出，作为Decoder的输入，但是是在Decoder的两个Attention层之间进行输入。这个地方比较有意思了。

Decoder：前t−1t-1t−1时刻输入 + 位置编码 -> Mask 多头自注意力层 -> Add & Norm-> + Encoder output -> {多头自注意力层 -> Add & Norm -> MLP } -> Output。

用{}框起来的部分实际上就是一个Encoder，也就是说Decoder比Encoder多一个Mask-MH-Atten+Add&Norm，这部分是处理t−1t-1t−1时刻输出的token，经过MMHA处理后，作为v进入下一部分的MHA，以及Add & Norm的残差块。

Encoder和Decoder块

Encoder：用N=6的完全一样的层，每个层里面有两个子层，即多头注意力层和()MLP层，对每一个子层用一个残差连接。最后用一个Layer Normalization。

用公式表示每个子层：LayerNorm(x+Sublayer(x))LayerNorm(x+Sublayer(x))LayerNorm(x+Sublayer(x))

为了构建这些残差连接，设定embedding的维度是512。

BatchNorm：在一个mini-batch里，计算出均值和方差，然后归一化使得均值为0，方差为1。注意在训练的时候还是要把全局均值和方差存起来。

同时，还可以把每个batch的均值和方差映射到任意值。

BatchNorm是针对每一个特征做的。

LayerNorm：对每个样本做的。在二维的情况下，可以认为是(LayerNorm(XT))T(LayerNorm(X^T))^T(LayerNorm(XT))T。

但是在Transformer里是针对序列做的，一个样本里有很多个元素，实际上是三维样本：(batch, seq(n), feature(512))，长、宽、深。

BatchNorm相当于垂直于feature、平行于seq切一刀。注意每个句子的seq是不一样长的，如果预测阶段碰到了特别长的seq（训练样本都没有那么长），那么计算出来的全局均值和方差可能就没有那么好用。

LayerNorm相当于垂直于batch、平行于feature切一刀。LayerNorm的情况比BatchNorm要好一些，每个seq计算自己的均值和方差，也不需要存一个全局的均值和方差。因为是对每个样本来做的，无论单个样本是长还是短，影响不会太大。

之后对LN的研究主要是用LN可以对收敛有影响（Lipschitz常数更大）

Decoder：同样是N=6个堆叠起来，但是用了三个子层，同样是LayerNorm。但是解码器用了自回归模型。由于自注意力机制可以看到整个序列的信息，因此在预测第ttt个词的时候，需要用掩码自注意力，保证训练在第ttt时间，看不到后面的词。

注意力

注意力函数是将一个query和一些key-value对映射成一个输出的函数。

q、k、v、output都是一些向量。output是value的加权和，所以这两者的维度一样。

这个权重是通过计算查询的query和value对应的key的相似度得到的，用了compatibility function。

缩放点积自注意力 Scaled Dot-Product Attention

Attention(X)=softmax(QKTdk)VAttention(X) = softmax(\frac{QK^T}{\sqrt{d_k}})V Attention(X)=softmax(dk​​QKT​)V

Q、K、V的长度都是dkd_kdk​，Q和K两个向量做内积算相似度（余弦相似度）

放进softmax就得到n个非负的，加起来等于1的向量，作为权重。

然后把这些权重作用在V上。

Q(n×dk)Q(n \times d_k)Q(n×dk​)， K(m×dk)K(m \times d_k)K(m×dk​)， QKT(n×m)QK^T(n \times m)QKT(n×m)

Softmax是对每一行做Softmax。

V(m×dv)V(m \times d_v)V(m×dv​)

Attention(X)：(n×dv)Attention(X)：(n \times d_v)Attention(X)：(n×dv​)，每一行就是需要的输出。

加性注意力：可以处理Q和K长度不一样的情况。

点积注意力：和上面的差不多，但是不用除以dkd_kdk​。

当dkd_kdk​不是很大的时候，除不除都没有关系。

当dkd_kdk​很大的时候，点积的值可能会很大/小，最大的那个值做softmax的时候会更靠近1，最小的softmax值会更靠近0（即值会更向两端靠拢）。

当出现这样的情况，计算梯度的时候，梯度会比较小。因为Softmax是置信的地方更靠近1，不置信的地方更靠近0，所以这时候网络就跑不动了。

在Transformer里的dkd_kdk​比较大，所以除以dk\sqrt{d_k}dk​​是个不错的选择。

如何做Mask呢？

假设Q和K长度均为n，并且在时间上都可以对应起来。

在第ttt时刻，QQQ应该只能看到{q1,q2,...,qt−1}\{q_1, q_2, ..., q_{t-1}\}{q1​,q2​,...,qt−1​}，而不应该看到{qt,...,qn}\{q_t, ..., q_n\}{qt​,...,qn​}。

但是实际计算的时候，是可以看到整个Q的。

Maske就是把{t,..,n}\{t,.., n\}{t,..,n}的Q值乘以一个很大的负数（比如−106-10^{6}−106），这样进入Softmax的时候，这个对应的q值就会趋近于0。

为什么不能直接置0呢？如果直接弄成0，那权重之和就不是1了。

多头注意力

与其做一个单独的注意力，不如把整个query、key和value投影到一个低维，投影h次。然后再做h次的注意力函数。最后再合并起来、再做一次线性的投影。

为什么要做多头注意力机制呢？

为了识别不一样的模式、用一些不一样的计算相似度的办法。

前面的缩放点积注意力其实没有什么可以学的东西，但是这里的低维投影是可以学的，相当于我给你h次机会，希望你能学到点不一样投影的方法。

这里有一点点像卷积网络里面有多个输出通道的意思。

MultiHead(Q,K,V)=Concat(head1,...,headh)WoMultiHead(Q,K,V) = Concat(head_1, ..., head_h)W^oMultiHead(Q,K,V)=Concat(head1​,...,headh​)Wo

where headi=Attention(QWiQ,KWiK,VWiK)head_i=Attention(QW_i^Q, KW_i^K, VW_i^K)headi​=Attention(QWiQ​,KWiK​,VWiK​)

WiQ∈Rdmodel×dkW_i^Q \in \mathbb{R}^{d_{model} \times d_{k}}WiQ​∈Rdmodel​×dk​

WiK∈Rdmodel×dkW_i^K \in \mathbb{R}^{d_{model} \times d_k}WiK​∈Rdmodel​×dk​

WiV∈Rdmodel×dvW_i^V \in \mathbb{R}^{d_{model} \times d_v}WiV​∈Rdmodel​×dv​

特别的，Wo∈Rhdv×dmodelW^o \in \mathbb{R}^{hd_v \times d_{model}}Wo∈Rhdv​×dmodel​

在Transformer原始工作里，h=8，设定dk=dv=dmodelh=64d_{k} = d_v = \frac{d_{model}}{h} = 64dk​=dv​=hdmodel​​=64

Transformer的应用

编码器的自注意力在做什么？

对于输入X（进入MHA分别为Key、Value和Query，同样的X复制成三份），输入n个Query就有n个Output，并且Ouput长度维度不变。

不考虑多头的情况，encoder的输出就是对输出的加权和，而权重来自于自己本身。

多头的话，因为有投影，因为会学习h个东西出来。

解码器不一样的地方，一个是有Masked这个东西。

另一个是后面的MHA不再是自注意力，Key和Value是来自Encoder的output，而Query是来自Masked的输出。

用Maske出来作为Query，计算Encoder出来的Key-Value的加权和，这个权重的粗细力度。
